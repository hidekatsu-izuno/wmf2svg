# wmf2png Native Save Fallback

## Purpose
Rework `src/test/bin/wmf2png.ps1` so WMF-to-PNG output follows the renderer comparison results: use GDI+ native `Image.Save(PNG)` when it matches mspaint behavior, and keep explicit canvas rendering where native save would produce the wrong size.

## Context
- `2264z_01_1.wmf` matched mspaint bit-for-bit when converted by GDI+ `Image.Save(PNG)`.
- `2doorvan.wmf` did not work as a general direct-save replacement: native save produced `318x116` instead of the expected `477x173`.
- The safe rule is to use native save only when:
  - `-Transparent` is requested,
  - the WMF is non-placeable,
  - the resolved output dimensions equal the GDI+ image's native dimensions.
- For placeable WMFs and explicitly resized output, keep the existing explicit canvas path.

## Summary
- Removed the dedicated GDI metafile playback path from `wmf2png.ps1`; comparison showed it was not the closest result for `2264z_01_1`.
- Added a guarded native `Image.Save(PNG)` path for transparent, non-placeable WMFs when the resolved output size exactly matches the GDI+ native image size and the default DPI is used.
- Kept the explicit GDI+ `DrawImage` canvas path for placeable WMFs, explicit sizing, non-default DPI, and cases where native save would change dimensions.
- Verified `2264z_01_1.wmf -Transparent`: `4625x2358`, mspaint baseline `AE=0`, MD5 match.
- Verified `2doorvan.wmf -Transparent`: `477x173`, mspaint baseline `AE=0`.
- Verified `123.wmf -Transparent`: `3870x4631`, mspaint baseline `AE=0`, MD5 match.

## Goals
- `2264z_01_1.png` generated by `wmf2png.ps1 -Transparent` should match the mspaint baseline with `AE=0`.
- `2doorvan.png` should keep the expected `477x173` dimensions.
