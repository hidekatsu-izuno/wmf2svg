# Filter Font Emheight To GDI Face Names

## Purpose

Keep `font-emheight.*` entries only for font names that GDI exposes as `LOGFONT.lfFaceName` candidates, instead of filtering by style words alone.

## Context

- The user clarified that entries should remain when the name can be referenced as `lfFaceName`.
- `System.Drawing.Text.InstalledFontCollection` can expose family/style names differently from GDI face enumeration.
- GDI `EnumFontFamiliesEx` provides the authoritative installed face names for `lfFaceName` matching.
- The worktree has unrelated existing edits in `FontUtil.java`, `FontUtilTest.java`, and `SvgGdi.properties`; preserve unrelated changes.

## Summary

Implemented the corrected rule: keep `font-emheight.*` entries when the name is
both:

1. emitted by `src/test/bin/fontmetrics.ps1`, and
2. enumerated by GDI `EnumFontFamiliesExW` as `LOGFONT.lfFaceName`.

Temporary GDI enumeration found 341 current face names. The `fontmetrics.ps1`
output contained 249 non-default-ratio entries, and all 249 were exposed by GDI
as face names.

Replaced `SvgGdi.properties` `font-emheight.*` entries with those 249 desired
entries. This restores valid GDI face names that contain style words, such as:

- `Arial Black`
- `Arial Rounded MT Bold`
- `Segoe UI Semibold`
- `Noto Sans JP Light`

It also removes non-current/non-GDI-face `font-emheight.*` entries such as
`Academy Engraved LET` from the current file.
